# 私有化安全即时通讯系统功能说明书

## 1. 项目定位

本系统是面向企业、团队和私有组织的安全即时通讯平台，目标是快速上线、支持私有化部署、支撑几千到几万人稳定使用，并可持续二次开发。

系统基于 Matrix 开源生态建设，服务端采用 Synapse，客户端采用 Flutter + Matrix SDK 二开，后台采用独立管理系统封装 Synapse 管理能力。

本项目不是 safeX 官方源码，也不包含 safeX 私有代码。

### 2.2 即时通讯

### 2.3 群组能力

### 2.5 文件与媒体

- 图片发送

- 服务端媒体存储

- 媒体访问权限控制
- 按用户/群聊查看媒体

### 2.6 管理后台

- 管理员登录
- 用户管理
- 群组管理
- 文件/媒体管理
- 举报审核
- 系统公告
- 注册码/邀请码管理
- 基础统计
- 客户端配置
- 安全策略配置

后台核心原则：

- 运营人员只操作你的后台。
- 后台服务调用 Synapse Admin API。
- App 不直接调用 Synapse Admin API。
- Synapse 管理员 Token 只保存在后台服务端。
- 生产环境不把 `/_synapse/admin/*` 暴露到公网。

### 2.7 私有化部署

- Docker Compose 部署
- PostgreSQL 数据库
- Redis 缓存
- Nginx 入口代理
- coturn 音视频中继
- 环境变量配置

本地开发阶段：

- Synapse 使用 `http://localhost:8008`
- TURN 使用 `localhost:3478`
- 暂不绑定域名
- 暂不配置公网 TLS

## 7. 管理后台说明

管理后台不直接替代 Synapse，而是作为业务控制台封装底层管理能力。

后台角色：

- 系统所有者
- 管理员
- 审计员
- 客服支持

本地 Demo 已内置三种后台角色：

- `owner`：系统所有者，拥有全部后台能力。
- `admin`：运营管理员，可以执行用户、群聊、媒体、举报、通知、注册和维护写操作。
- `auditor`：审计员，只能查看数据和导出/筛选审计日志，不能执行写操作。

后台模块：

- 首页看板
- 用户管理
- 群组管理
- 文件管理
- 举报审核
- 公告管理

## 8. 合规与安全提醒

端到端加密开启后，服务端无法读取用户聊天明文。如果业务需要内容审计，必须在产品策略、用户协议和合规要求中提前设计，不能简单依赖服务端读取消息。

如果修改 Synapse 源码或将其作为商业服务提供，需要关注其开源许可证要求，并在商业化前进行法律审查。
